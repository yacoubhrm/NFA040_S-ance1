CD/CI - Intégration Continue et Déploiement Continu

L'intégration continue (CI) est une pratique de développement où les développeurs intègrent régulièrement leur code dans un dépôt partagé. Chaque intégration est vérifiée par une construction automatisée pour détecter les erreurs rapidement.

Le déploiement continu (CD) va un pas plus loin en automatisant le déploiement du code validé vers un environnement de production, ce qui permet de livrer de nouvelles fonctionnalités plus rapidement et avec plus de fiabilité.

Technologies Utilisées

Voici quelques-unes des technologies et outils que j'utilise régulièrement :

  • GitHub : Une plateforme de gestion de versions et de collaboration sur le code. Elle permet de gérer le code source avec Git et offre des fonctionnalités puissantes pour le CD/CI grâce à GitHub Actions.
  • Jenkins : Un serveur open source d'automatisation permettant d'automatiser les différentes étapes du processus de développement.
  • Docker : Un outil permettant de créer, déployer et exécuter des applications dans des conteneurs, assurant ainsi la portabilité et la cohérence entre différents environnements.
  • Kubernetes : Un système d'orchestration de conteneurs pour automatiser le déploiement, la gestion et la mise à l'échelle des applications conteneurisées.
